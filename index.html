Introduction

[↑]

A qui s'adresse ce cour ?

A tout ceux qui souhaite mettre un peu d'interactivité dans leur site internet, aborder la programmation JavaScript en douceur ou qui souhaite faire des animations sur mesure.

Et il y a-t-il des prérequis ?

Il faut absolument avoir une bonne compréhension de HTML5 et CSS3 qui constitue le cadre d'une page HTML. Ensuite parce que jQuery intégre beaucoup d'élément de CSS3 dans sa façon de filtrer ou de sélectionner les éléments de la page.

Cette première partie aborde les bases (events, contenu, CSS), les effets graphiques (show/hide, fade, slide, toggle, animate) et plusieurs concepts intermédiaires (get / set, callback, chainage, filtrage, pacourir le DOM).

Une deuxième partie sera consacré à Ajax dans jQuery dans un contexte de réalisation de site Web Mobile et Web App non native. On abordera les problèmes liés au Responsive Web Design ainsi qu'au développement Mobile et son optimisation.

2.3 Usage de jQuery

  1. La libraie jQuery doit toujours être chargé avant votre code, sinon ça marche pas :-D
  2. L'attibut type= “text/javascript” n'est pas nécessaire en HTML5, on peut l'utiliser quand même pour rendre le code plus lisible, si ça vous chante.

Si on lie la librairie (en direct ou par CDN) dans le <head> de la page on doit attendre que le chargement de la page soit terminé pour lancer nos scripts. Pour cela on utilise une méthode simple : 

<script type="text/javascript">
	$(document).ready(function(){

		// mon code ici
	});
</script>
  • $ correspond à l'appel de la librairie, c'est 'équivalent de jQuery.
  • Puis on cible document suivie de ready, ce qui signifie quand le document (la page) est prêt, sous entendu chargé.
  • Alors la function lance le script.

On doit toujours placer son code à l'intérieur de ce genre de lanceur (Ready Event) ; il y en a avec d'autres méthodes, celle-ci est la plus courante.

voir l'exemple-001 example-001.html

<script type="text/javascript">
	$(document).ready(function(){
		$("h1").css("color", "#0088ff");
	});
</script>

On change la couleur du titre avec un bout de code : voir l'exemple-002 example-002.html

  • Appel de jQuery avec le - $()
  • On cible la balise h1 - $("h1")
  • la méthode css() permet d'ajouter une règle de style a la cible (et donc à h1) - $("h1").css()
  • on passe en paramètre à la méthode css() la propriété et la valeur correspondant à la règle css.
    Ici c'est la propriété color et la valeur #0088FF pour un bleu clair.

3. Syntaxe de jQuery

[↑]
  • On charge la librairie depuis un lien local ou depuis le réseau via un CDN.
  • La librairie est toujours chargé en premier, avant le code personnel.
  • Le chargement se fait entre des balises <script>
    • soit dans le <header>
    • soit en bas de page juste avant la balise de fermeture du </body>
  • On peut écrire les scripts personnels dans la page ou dans un page extérieure qui est chargé après la librairie.
  • Un Ready Event doit toujours préparer le chargement de la page $(document).ready(handler)
  • Les déclarations jQuery commencent généralement par un $ (signe dollar) et se terminent par; (un point-virgule).
  • Les instructions jQuery utilisent des gestionnaires d'évènement (Event Handler) $(sélecteur) .action ();

 

$("li.hot").addClass("furax");
  • L'objet jQuery : $()
  • L'objet jQuery et son sélecteur : $("li.hot")
  • L'opérateur qui attache la méthode à l'objet : . (un point)
  • La méthode : addClass()
  • La méthode avec son paramètre : addClass("furax"), dans ce cas le nom d'une classe CSS.
  • On termine par un ; point virgule.

 

3.1 Le gestionnaire d'évènement

Pour effectuer des actions on utilise des gestionnaires d'évènements (Event Handler) avec jQuery. On pointe un élément appellé sélecteur sur lequel on applique une action (méthode) comme ceci : $(sélecteur).action()

On change le texte du paragraphe : voir l'exemple-003 example-003.html

$("p").text("Hello World!");
  • $("p") sélectionne toutes les balises <p> de la page.
  • La méthode text() permet d'insérer du texte.

Le problème c'est que c'est fait juste après le chargement de la page. Essayons ajouter un évènement.

On ajoute un bouton pour provoquer le remplacement du texte par l'utilisateur : voir l'exemple-004 example-004.html

$("button").click(function(){
	$("p").text("Hello World!");
});
  • $("button") sélectionne toute les balises <button>
  • On y applique la méthode .click()
  • Cette méthode lance une fonction générique qui contient :
  • $("p") le sélecteur de toutes les balises <p> de la page.
  • Sur ce sélecteur on applique la mééthode .text() qui ajoutera le texte qu'il contient.

Mais il ajoutera le texte qu'il contient seulement au click sur le bouton. Remarquez comme les instructions sont imbriqués, l'action est dans le bouton.

4. Sélecteurs jQuery

[↑]

L'utilisation la plus courante de JavaScript est de modifier des éléments HTML de la page. Il s'agit de sélectionner ou de créer des éléments puis de leurs appliquer des effets tel que afficher / masquer, animer ... jQuery facilite une sélection rapide des éléments HTML du DOM.

Donc avant d'appliquer un quelconque effet ou une action, vous devez attraper votre cible. Cela peut être une ID, une Class, une balise, un attribut de balise, tout ça à la fois ou plus.

 

4.1 Sélection par l'ID

On utilise une ID quand on cherche à sélectionner un élément unique, car les ID sont toujours unique. Les ID sont généralement réservé pour l'usage de JavaScript dans les méthodes moderne de CSS.

<script type="text/javascript">
	$(document).ready(function(){
		// Highlight element with id mark
		$("#mark").css("background", "yellow");
	});
</script>

On reconnait le Ready Event qui préfigure chaque script dans une page.

  • $('#mark') vous noterez les guillemets pour le teste et le dièze (#) pour l'ID notre cible !
  • auquel on applique la méthode css()
  • avec les paramètres : propriété color, valeur yellow (notez les guillemets hein!).

Voir la page d'exemple : Example-005.html

 

4.2 Sélection par nom de classe

La sélection par nom de classe se fait sur le ou les éléments possédant la classe spécifique.

<script type="text/javascript">
	$(document).ready(function(){
		// Highlight element with class mark
		$(".mark").css("background", "yellow");
	});
</script>

Toujours le Ready Event qui lance le code au chargement de la page.

  • $('.mark') notez les guillemets et le point devant le nom de la classe cible.
  • auquel on applique la méthode css()
  • avec les paramètres : propriété color, valeur yellow (notez les guillemets).

Voir la page d'exemple : Example-006.html

 

4.3 Sélection par le nom de balise

On peut simplement sélectionner un élément sur son nom de balise.

<script type="text/javascript">
	$(document).ready(function(){
		// Highlight element with p tag
		$('p').css("background", "yellow");
	});
</script>

Toujours le Ready Event , oui je sais je saoule.

  • $('input[type="text"]') notez les guillemets simple et double et la sélection d'attribut comme dans CSS.
  • auquel on applique la méthode css()
  • avec les paramètres : propriété color, valeur yellow (notez les guillemets).

Voir la page d'exemple : Example-007.html

 

4.4 Sélection par l'attribut

On peut sélectionner des éléments an se basant sur la valeur de leurs attributs.

<script type="text/javascript">
	$(document).ready(function(){
		// Highlight element with attribute tag
		$('input[type="text"]').css("background", "yellow");
	});
</script>

Toujours le Ready Event , oui je sais...

  • $('input[type="text"]') notez les guillemets simple et double et la sélection d'attribut comme dans CSS.
  • auquel on applique la méthode css()
  • avec les paramètres : propriété color, valeur yellow (notez les guillemets).

Voir la page d'exemple : Example-008.html

 

4.5 Sélection composée

Les sélecteurs peuvent également être combinés pour rendre les choses plus précises. Un sélecteur de classe peut être combiné à un sélecteur d'élément pour rechercher des combinaisons spécifiques. 

<script type="text/javascript">
	$(document).ready(function(){
		// Highlight paragraph elements with class mark
		$("p.mark").css("background", "blue");
		// Highlight span elements inside the element with ID mark
		$("#mark span").css("background", "green");
		// Highlight li elements inside the ul elements
		$("ul li").css("background", "red");
		// Highlight li elements inside the ul element with id mark
		$("ul#brand li").css("background", "yellow");
		// Highlight a particular li element
		$("ul li.brand").css("background", "silver");
		// Highlight all anchor elements with target blank
		$('a[target="_blank"]').css("background", "orange");
	});
</script>

Voir la page d'exemple : Example-009.html

 

4.6 Sélecteur de jQuery

En plus des sélecteurs définis par CSS, il existe également des sélecteurs personnalisés dans jQuery. 

<script type="text/javascript">
	$(document).ready(function(){
		// Highlight table rows appearing in odd places
		$("tr:odd").css("background", "yellow");
		// Highlight table rows appearing in even places
		$("tr:even").css("background", "orange");
		// Highlight first paragraph element
		$("p:first").css("background", "red");
	});
</script>

Ici on utilise impair (odd) et pair (even) et le premier (first).

Voir la page d'exemple : Example-010.html

5.1 Evènement de souris (Mouse Event)

Nous allons voir les évènements de souris les plus courants.

 

5.1.1 La méthode click()

Cette méthode est déclenché par le click de l'utilisateur. 

<script type="text/javascript">
	$(document).ready(function(){
		$("p").click(function(){
			$(this).slideUp();
		});
	});
</script>
  • On applique la méthode click() sur l'élément <p>
  • La fonction générique de click() lance la suite,
  • $(this) est une variable qui permet de retourner l'objet à lui même, ici $('p').click().
  • Et donc de lui appliquer une nouvelle méthode, ici slideup() (littéralement "glisse vers le haut").

Voir la page d'exemple : Example-012.html

 

5.1.2 La méthode dblclick()

Méthode identique à la précédente en dehors du double click au lieu du simple click.

<script type="text/javascript">
	$(document).ready(function(){
		$("p").dblclick(function(){
			$(this).slideUp();
		});
	});
</script>

Voir la page d'exemple : Example-013.html

 

5.1.3 La méthode hover()

Cette méthode associe souvent deux fonctions de gestion d'élément. Dans notre exemple on ajoute une classe CSS au survol et on la retire à la fin. 

	<script type="text/javascript">
		$(document).ready(function(){
			$("p").hover(function(){
				$(this).addClass("highlight");
			}, function(){
				$(this).removeClass("highlight");
			});
		});
	</script>
  • On cible la balise <p> avec $('p')
  • On lui associe la méthode de détection de survol hover()
  • Cette méthode produit une première fonction générique
    • grâce à la variable $(this) on applique la méthode addclass() sur $('p').hover()
  • Cette méthode produit une seconde fonction générique
    • grâce à la variable $(this) on applique la méthode removeclass() sur $('p').hover()

C'est très amusant à regarder dans l'inspecteur de votre navigateur, on voit apparaître et disparaître la classe highlight. [images/img-001.png ]

Voir la page d'exemple : Example-014.html

 

5.1.4 La méthode mouseenter()

Cette méthode attache un gestionnaire d'évènement à un élément donné. Ici le fait que la souris soit sur l'élément. 

<script type="text/javascript">
	$(document).ready(function(){
		$("p").mouseenter(function(){
			$(this).addClass("highlight");
		});
	});
</script>
  • On cible la balise <p> avec $('p')
  • On lui applique la méthode mouseenter() qui détecte les survol de souris.
  • On ajoute une fonction générique pour activer l'évènement
  • On utilise la variable $(this) pour appliquer la méthode addclass() à $('p').mouseenter()
  • La méthode addclass() permet d'ajouter un attribut class à la balise <p> et d'insérer la classe highlight.

Voir la page d'exemple : Example-015.html

 

5.1.5 La méthode mouseleave()

Cette méthode attache un gestionnaire d'évènement à un élément donné et complète la précédente. Ici le fait que la souris soit sortie de l'élément, quelle l'ait quitté. 

<script type="text/javascript">
	$(document).ready(function(){
		$("p").mouseleave(function(){
			$(this).removeClass("highlight");
		});
	});
</script>
  • On cible la balise <p> avec $('p')
  • On lui applique la méthode mouseleave() qui détecte la sortie du survol de souris.
  • On ajoute une fonction générique pour activer l'évènement.
  • On utilise la variable $(this) pour appliquer la méthode removeclass() à $('p').mouseleave()
  • La méthode removeclass() permet de supprimer toutes les classes présente dans l'attribut class de la balise <p>.

Voir la page d'exemple : Example-016.html

 

5.2 Evènement de clavier (Keyboard Event)

Les évènements de clavier sont activés quand l'utilisateur appuie ou relâche une touche du clavier.

 

5.2.1 La méthode keypress()

Cette méthode détecte si une touche a été enfoncée. 

<script type="text/javascript">
	$(document).ready(function(){
		var i = 0;
		$('input[type="text"]').keypress(function(){
			$("span").text(i += 1);
			$("p").show().fadeOut();
		});
	});
</script>
  • On déclare la variable i et on lui affecte la valeur de zéro.
  • On cible le champ de texte $('input[type="text"]')
  • On lui applique la méthode keypress() qui détecte quand on enfonce une touche.
    • La fonction générique lance la suite
  • On cible le span $("span") pour lui injecter du texte
    • Ce texte n'est autre que la variable i qu'on additionne à elle même
    • i += 1; c'est l'équivalent de i = i + 1;
  • Puis on applique les effets graphique sur <p> avec les mérhodes show() et fadeOut().

La méthode keypress() ressemble beaucoup à la méthode keydown() à une exception près, les touches modifiées et non-imprimables (Esc, Maj, Suppr, Retour, les flèches ...) ne déclencheront pas l'évènement.

Voir la page d'exemple : Example-017.html

 

5.2.2 La méthode keydown()

Cette méthode détecte si n'importe quelle touche a été enfoncée. 

	$(document).ready(function(){
		var i = 0;
		$('input[type="text"]').keydown(function(){
			$("span").text(i += 1);
			$("p").show().fadeOut();
		});
	});
</script>

Même méthode que keypress(), mais celle-ci détecte toutes les touches.

5.3 Evènement de formulaire (Form Event)

Les évènements de formulaire sont déclanchés lorsque les contrôles de formulaire sont activés ou perdus, lorsqu'une valeur de contrôle de formulaire est modifié par l'utilisateur (entrée un texte par exemple). Les évènements de formulaires les plus courants sont les suivant :

 

5.3.1 La méthode change()

Cette méthode est associé aux éléments <textarea><input><select>. Elle sera activé quand la valeur de l'élément changera. Choisissez un rayon dans le panier :-D

<script type="text/javascript">
	$(document).ready(function(){
		$("select").change(function(){
			var selectedOption = $(this).find(":selected").val();
			alert("Vous avez sélectionné - " + selectedOption);
		});
	});
</script>
  • On cible la balise <select> $("select") et on lui applique la méthode change()
    • Au premier changement elle lance la fonction générique
  • On créé une variable selectedOption à laquelle on affecte :
    • $(this) c'est l'objet en cours, c'est à dire $("select").change()
    • sur lequel on applique la méthode find() pour qu'elle cherche la valeur "selected" et donc bien celle qui vient d'être sélectionné dans le menu déroulant.
  • En sortie on utilise la méthode alert() qui créé une fenêtre dans laquelle on écrit "Vous avez sélectionné - " + notre variable selectedOption, c'est à dire la valeur sélectionné dans la liste.
  • C'est nettement plus compliqué à expliquer qu'à faire :-D

Voir la page d'exemple : Example-020.html

La méthode focus() attache un gestionnaire d'évènement à des éléments selectionnés, c'est-à-dire pointé par défaut ou par l'utilisateur. 

<script type="text/javascript">
	$(document).ready(function(){

			$(this).next("span").show().fadeOut("slow");
		});

</script>
  • On cible les champs texte $("input") en lui appliquant la méthode focus().

La méthode blur() est complémentaire de la méthode focus() puisqu'elle ne s'active que lorsque le focus est perdu. Généralement associé aux évènements de formulaire avec des balises telles que <textarea> <input> <select>.

5.3.4 La méthode submit()

Cette méthode permet de contrôler si l'utilisateur tente de soumettre le formulaire ce qui permet de pouvoir y attacher des fonctions de contrôle le plus souvent. 

<script type="text/javascript">
	$(document).ready(function(){
		$("form").submit(function(event){
			var regex = /^[a-zA-Z]+$/;
			var currentValue = $("#firstName").val();
			if(regex.test(currentValue) == false){
				$("#result").html('<p class="error">Not valid!</p>').show().fadeOut(1000);
				// Preventing form submission
				event.preventDefault();
			}
		});
	});
</script>
  • On cible le formulaire $('form') sur lequel on applique la méthode submit().
    • Dans la fonction générique on prend soin de mettre event en paramètre puisqu'on attend que le formulaire soit soumis (évènement de soumission donc).
  • On créé une variable regex avec un filtre d'expression régulière pour filtrer les lettres minuscule et majuscule.
  • Puis une seconde variable currentValue qui cible l'ID $("#firstName") ou plutôt sa valeur val().
  • On fait un test (méthode test()) sur la valeur de l'ID FirstName pour savoir si elle correspond au filtre de la variable regex (c'est-à-dire des lettres minuscule ou majuscule).
    • Si le test répond par true, tout va bien.
    • Si le test répond par false :
    • On cible l'ID result $("#result") et on applique la méthode html() pour écrire du HTML.
    • Sur ce texte html on ajoute les méthodes show() et fadeOut() pour le faire apparaître et disparaître au bout de 1000 ms soit 1 seconde.

9. Effets d'animation

Comment utiliser jQuery pour animer les propriétés CSS.

 

9.1 La méthode animate()

La méthode animate() permet de créer des animations personnalisées. Elle est utilisé pour manipuler des propriétés CSS comme height, width, padding, margin, top, left, opacity ... Mais toutes les propriétés CSS ne sont pas prise en charge par jQuery comme les couleurs de fond (il y a un plugin).

API Docs: https://api.jquery.com/animate/#animate-properties-duration-easing-complete

La syntaxe

$(selector).animate({ properties }, duration, callback);

Notez bien les accolades autour des propriétés, les virgules séparant les paramètres.

  • selector : c'est la cible bien sûr, sur laquelle on applique l'animation.
  • animate() : notre méthode.
  • properties : la ou les propriétés CSS à animer.
  • duration : le paramètre de durée (facultatif). Il est spécifié en milliseconde ou avec une chaîne prédéfinie comme slow ou fast.
  • callback : la fonction de rappel est facultative. Mais à quoi ça sert un callback ? Et bien à appeller une fonction à la fin de l'animation, ça peut être utile :-) (voir Chap. 10)

Ci-dessous une petite animation qui déplace une image de 300px vers la droite.

<script type="text/javascript">
	$(document).ready(function(){
		$("button").click(function() {
			$("img").animate({
				left: 300
			});
		});
	});
</script>

En HTML tous les éléments ont une position statique (static) par défaut. Pour pouvoir déplacer un élément il faut donc le définir dans une position relative, fixe ou absolue sinon ça ne bougera pas.

Voir la page d'exemple : Example-046.html

 

9.2 Animation simultanée de plusieurs propriétés

Il est possible d'animer plusieurs propriétés d'un élément simultanément avec la méthode animate().

<script type="text/javascript">
	$(document).ready(function(){
		$("button").click(function(){
			$(".box").animate({
				width: "300px",
				height: "300px",
				marginLeft: "150px",
				borderWidth: "10px",
				opacity: 0.5
			});
		});
	});
</script>

Ici toutes les animations sont animées en même temps et sans délais.

ATTENTION : le nom des propriétés doivent être écrit en lower camel (camelCase), c'est-à-dire fontSize au lieu de font-size, marginRight au lieu de margin-right ou borderWith au lieu de border-with ... Ne confondez pas avec le CSS.

Ensuite on ne peut pas animer une propriété CSS qui n'aurait pas été définie avant. Si votre élément n'a pas de bordure définie en CSS, cette propriété n'existant pas elle ne pourra pas être animé par jQuery. Vous pouvez simplement la définir par défaut : border:none; ça suffit.

9.3.1 La méthode stop()

La méthode stop() permet d'arrêter une animation avant sont achèvement.

La syntaxe

$(selector).stop(stopAll, goToEnd);

  • selector : la cible de l'animation.
  • stop() : notre méthode.
  • stopAll : paramètre facultatif, permet de préciser si les animation en file d'attente doivent être supprimées ou non. Par défaut ce paramètre est sur false, ce qui signifie que seule l'animation en cours sera arrêtée, les autres s'exécuteront jusqu'à la fin.
  • goToEnd : par défaut ur false, permet d'indiquer si l'animation doit être terminée imédiatement. 
<script type="text/javascript">
	$(document).ready(function(){
		// Start animation
		$(".start-btn").click(function(){
			$("img").animate({"left": "+=150px"}, 2000);
		});
		// Stop running animation
		$(".stop-btn").click(function(){
			$("img").stop();
		});
		// Start animation in the opposite direction
		$(".back-btn").click(function(){
			$("img").animate({left: "-=150px"}, 2000);
		});
		// Reset to default
		$(".reset-btn").click(function(){
			$("img").animate({left: "50"}, "fast");
		});
	});
</script>

Voir la page d'exemple : Example-051.html

La méthode stop() fonctionne pour tous les objets jQuery (slide, fade, show, hide) ainsi que pour les animations personnalisées. Ci-dessous on applique une méthode stop() sur une méthode slideToggle(). 

<script type="text/javascript">
	$(document).ready(function(){
		// Kill and toggle the current sliding animation
		$(".toggle-btn").on("click", function(){
			$("img").stop().slideToggle(2000);
		});
	});
</script>

Voir la page d'exemple : Example-052.html

Lisser un effet de survol

Quand on a un effet de survol animé avec plusieurs animations en file d'attente qui doivent se déclanchées successivement alors que le curseur de la souris a un mouvement rapide. Cela peut interrompre une animation et faire un mauvais effet graphique. Pour empêcher ce genre de chose et créer un effet de survol plus doux et moins heurté on utilise la méthode stop(true, true). Cette méthode effacera toutes les animations en file d'attente et passera directement à la valeur finale. 

<script type="text/javascript">
	$(document).ready(function(){
		$(".box").hover(function(){
			$(this).find("img").stop(true, true).fadeOut();
		}, function(){
			$(this).find("img").stop(true, true).fadeIn();
		});
	});
</script>

Non ce n'est pas magique, souvenez vous de la syntaxe et particulièrement des paramètres de stop() : stopAll et goToEnd. Ces paramètres sont sur false par défaut mais si on les mets sur true ça donne :

  • arrête tout (stopAll) sous entendu toutes les animations en file d'attente, et
  • va à la fin (goToEnd) donc directement à la dernière animation.

10. Fonction de rappel (callback)

[↑]

Toutes les instructions sont exécuter en jQuery ligne par ligne (séquentiel). Mais les effets peuvent prendre du temps et une animation peut commencer alors que la première est en cours d'exécution. On peut arrêter cela grâce à la fonction de rappel. La fonction de rappel est exécuté dès que l'effet est terminé, elle est toujours le dernier argument de la méthode.

Par exemple sur la méthode slideToggle() : $(selector).slideToggle(duration, callback);

Illustrons le par un exemple ci-contre. Ce code entraine l'affichage immédiat de l'alerte, il n'attendra pas que l'effet de bascule soit terminé. 

<script type="text/javascript">
	$(document).ready(function(){
		$("button").click(function(){
		$("p").slideToggle("slow");
			alert("The slide toggle effect has completed.");
		});  
	});
</script>

Voir la page d'exemple : Example-057.html

Dans l'exemple suivant on voit bien que la fonction de rappel (callback) est appelé seulement quand tous les effets sont terminés par opposition à tout en une fois. 

<script type="text/javascript">
	$(document).ready(function(){
		$("button").click(function(){
			$("p").slideToggle("slow", function(){
				// Le code est exécuté une fois l'effet terminé.
				alert("The slide toggle effect has completed.");
			});
		});
	});
</script>

Voir la page d'exemple : Example-058.html

Si vous appliquez la méthode à plusieurs éléments, la fonction de rappel sera appelée une fois pour chacun des éléments spécifiés. Toujour le même exemple mais avec une méthode appliqué à <h1> et <p>. 

<script type="text/javascript">
	$(document).ready(function(){
		$("button").click(function(){
			$("h1, p").slideToggle("slow", function(){
				// Code to be executed once effect is complete
				alert("The slide toggle effect has completed.");
			});
		});
	});
</script>

11 Getters / Setters

[↑]

Nous allons maintenant voir comment utiliser jQuery pour obtenir ou définir le contenu et les valeurs d'attribut d'un élément, ainsi que la valeur du contrôle de formulaire.

Certaines méthodes de jQuery peuvent être utilisées pour attribuer (set) ou lire (get) des valeurs d'éléments sélectionnés. Cela inclus les méthodes html(), text(), val(), attr().

  • Lorsque nous appelons ces méthodes sans utiliser d'arguments elles permettent d'obtenir ou de lire la valeur de l'élément sélectionné (GET).
  • Quand nous les appelons avec une valeur, elles vont assigner (SET) la (nouvelle) valeur.

 

11.1 La méthode text()

Dans l'exemple suivant nous allons récupéré (GET) le contenu des paragraphes.

<script type="text/javascript">
	$(document).ready(function(){

		$(".btn-one").click(function(){
			var str = $("p").text();
			alert(str);
		});
		// Récupère le contenu du premier paragraphe
		$(".btn-two").click(function(){
			var str = $("p:first").text();
			alert(str);
		});
	}) ;
</script>

Notez que la méthode text() n'a pas de valeur, donc là on est bien en position de lire (GET) les valeurs des paragraphes.

Voir la page d'exemple : Example-060.html

A présent nous allons insérer (SET) du texte dans les paragraphes. 

<script type="text/javascript">
	$(document).ready(function(){
		// Set text contents of all paragraphs
		$(".btn-one").click(function(){
			$("p").text("This is demo text.");
		});
		// Set text contents of the first paragraph
		$(".btn-two").click(function(){
			$("p:first").text("This is another demo text.");

12. Insérer du contenu

[↑]

jQuery donne un certain nombre de techniques pour ajouter du nouveau contenu dans un élément déjà existant, notamment :

  • append()
  • prepend()
  • html()
  • text()
  • before()
  • after()
  • wrap()
  • entre autre ...

Les méthodes text () et html () ont déjà été abordées, nous allons donc parler des autres.

 

12.1 La méthode append()

La méthode append() est utilisée pour insérer un nouveau contenu à la fin d'un élément.

<script type="text/javascript">
	$(document).ready(function(){
		// Append all paragraphs
		$("p").append(' <a href="#">read more...</a>');
			// Append an element with ID container
		$("button").click(function(){
			$("#container").append("This is demo text.");
		});
	});
</script>
  • Le premier append()
    • On cible les paragraphes de la page $("p")
    • Et on leur ajoute un lien "read more..." à la fin de la balise, après le texte.
  • Le second append()
    • Quand on clique sur le bouton cela ajoute du texte (demo) dans l'ID #container.
    • Ce conteneur (#container) est vide, mais théoriquement le texte est ajouté avant la balise de fermeture. Ajoutez du texte dans le conteneur et vous verrez bien !

Voir la page d'exemple : Example-069.html

 

12.2 La méthode preprend()

La méthode preprend() permet d'insérer du contenu après la balise ouvrante et donc avant le contenu existant. 

<script type="text/javascript">
	$(document).ready(function(){
		// Prepend all paragraphs
		$("p").prepend("<strong>Note:</strong> ");
		// Prepend an element with ID container
		$("button").click(function(){
			$("#container").prepend("This is demo text.");
		});
	});
</script>
  • Le premier prepend()
    • On cible les paragraphes de la page $("p")
    • Et on leur ajoute un texte "Note:" juste après la balise ouvrante et avant le texte.
  • Le second prepend()
    • Quand on clique sur le bouton cela ajoute du texte (demo) dans l'ID #container.
    • Ce conteneur (#container) est vide, mais théoriquement le texte est ajouté après la balise de d'ouverture. Ajoutez du texte dans le conteneur et vous verrez !

Voir la page d'exemple : Example-070.html

 

12.3 Insertions multiples avec append() et preprend()

Vous avez bien compris que prepend() place les éléments au début avant le contenu et append() après le contenu. Mais on peut aussi faire des insertions multiples.

$(selector).css("propertyName");

17. Parcourir le DOM

[↑]

Nous allons voir, à présent, comment parcourir le DOM grâce à jQuery.

Toutes les interactions des utilisateurs sur un site Web sont dictées par des événements. Le DOM a été standardisé par le W3C (UI Events). Chaque fois qu'un évènement (souris, clavier) se produit un objet (event / évènement) est créé. Le standard le divise en trois phases d'évènement :

  1. la phase de capture : l'objet évènement se propage par les ancêtres de la cible en partant de la fenêtre (Window) jusqu'aux parents (parent) de la cible.
  2. la phase de ciblage : l'objet évènement arrive sur sa cible et s'arrête.
  3. la phase à bulle : l'objet évènement se propage en ordre inverse des ancêtres de la cible en commançant par les parents (parent) et finissant par la fenêtre (window).

On ne va pas trop théoriser sur le fonctionnement des évènements, mais le parcours du DOM doit être en tête. Cet allez retour window --> document --> html --> body --> la cible et on remonte, vous ne devez pas l'oublier. Tous les évènements fonctionnent comme ça à moins d'être interrompu par un script. La traversée du DOM par jQuery est une fonctionnalité primordiale que l'on doit maîtriser pour en tirer parti.

Exemple d'arbre HTML :

<body>
	<div class="container">
		<h1>Hello World</h1>
		<p>This is an <em>simple paragraph</em>.</p>
		<ul>
			<li>Item One</li>
			<li>Item Two</li>
		</ul>
	</div>
</body>

On peut présenter le code ci-dessus comme ceci :

  • <body> est le parent de <div> et l'ancêtre de tout ce qu'il contient.
  • <div> est le parent de <p>, <h1>, <ul> ... C'est aussi l'enfant de <body>.
  • Les éléments <h1>, <p>, <ul> sont tous frères car ils sont sous le même parent.
  • <h1> est un enfant de <div> et également un descendant de <body>. il n'a pas d'enfant.
  • <p> est un parent de <em>, un enfant de <div> et un descendant de <body>.
  • <em> est également un descendant de <div> et <body>.
  • <ul> est le parent des éléments <li>, l'enfant de <div> et le descendant de <body>.
  • Les éléments <li> sont tous les enfants de <ul>, descendants de <body> et de <div> et de plus ils sont tous frères et soeurs (!).

Je pense que l'illustration généalogique est bien comprise ;-)

Maintenant que vous comprenez mieux les relations entre les éléments de l'arborescence du DOM, nous allons commencez à comprendre le fonctionnement de déplacement. Comment se déplacer latéralement, monter, descendre dans un arbre DOM avec jQuery ?

18.7 La méthode slice()

Avec la méthode slice() on peut couper le gâteaux en tranche (lol). Cette méthode accepte des numéros d'index de début et de fin comme argument. Le premier numéro donne la position d'où partir et le second la position où ça s'arrête (non inclu). 

<script type="text/javascript">
	$(document).ready(function(){
		$("ul li").slice(0, 2).addClass("highlight");
	});
</script>

Donc une petite tranchette de 0 à 2, soit 0, 1, on s'arrête à 2.

Voir la page d'exemple : Example-119.html

Essayons avec des valeurs négatives :

<script type="text/javascript">
	$(document).ready(function(){
		$("ul li").slice(-2, -1).addClass("highlight");
	});
</script>

Donc on part de la fin puisque la première valeur (-2) est négative, et plus précisément du deuxième en partant de la fin. Puis on va à -1 c'est-à-dire le premier élément en partant de la fin. Seul le 3ème élément sera sélectionné.
